# mdv

Markdown を快適にプレビューするための Electron デスクトップアプリ。Mermaid 図・コードハイライト・git 差分の可視化に対応。

## 機能

1. **Markdown 閲覧** — フォルダを開き、ツリーから `.md` / `.markdown` を選択して表示。GFM（表・チェックリスト等）、コードのシンタックスハイライト、Mermaid 図に対応。
2. **Mermaid ER図のテーブル名をダブルクリックで選択** — ER図のテーブル名（エンティティ名）をダブルクリックすると、その名前が選択状態になり手動で `Cmd+C` コピーできる。選択中のテーブルは枠線でハイライトされる。属性（カラム）行は対象外。
3. **開いているファイルの git 差分** — ツールバーの「☑ 差分」トグルで、いま見ている Markdown ファイル 1 つの HEAD からの変更を表示。Markdown をレンダリングしたまま、追加=緑・削除=赤取消線でインライン表示する（ブロック構造は保持）。Mermaid ER図は図の上でも差分を強調する（追加/変更テーブル・カラム=緑、削除があれば「変更前(HEAD)の図」を併記し削除=赤）。未コミット変更があるファイルは差分トグルにドットが付く。

その他:
- **自動更新** — 開いているファイルがディスク上で更新されると、表示・差分ともに自動で再読込（手動更新ボタンも残してあります）。
- **サイドバー折りたたみ** — ツールバー左の ☰ ボタンでファイルツリーを開閉。
- **ダークモード** — ツールバー右の ☾/☀ ボタンでライト/ダークを手動切替。Markdown 本文・コードハイライト・Mermaid 図・差分色まで連動し、選択は次回起動時も保持。

## セットアップ

### 前提
- **Node.js 18 以上**（推奨 20+）と npm
- **git**（git 差分機能を使う場合）

### 手順

```sh
# 1. クローン
git clone <このリポジトリのURL>
cd <クローンしたディレクトリ>

# 2. 依存をインストール
npm install

# 3. ビルド（out/ に成果物を生成）
npm run build

# 4. `mdv` コマンドを PATH に登録
npm link
```

これで、どのディレクトリからでも `mdv <ファイル|フォルダ>` が使えます。

```sh
mdv README.md   # 動作確認（このリポジトリの README を開く）
```

- アンインストール: `npm rm -g mdv`
- ソースを更新したら反映するには `npm run build`（`mdv` は `out/` のビルド成果物を使うため）

## 使い方（CLI）

`mdv <file|dir>` でビューアを直接起動できる。

```sh
mdv README.md      # ファイルを開く（親フォルダがツリーのルートになり自動選択）
mdv ./docs         # フォルダを開く
mdv                # 引数なし = カレントディレクトリを開く
```

- ファイル指定 → そのファイルの**親フォルダ**をルートにして、そのファイルを自動表示
- フォルダ指定 → そのフォルダをルートに表示
- ビルド成果物が無い状態で実行すると初回だけ自動ビルドする
- ターミナルはデタッチ起動のためすぐ戻る

## 開発モードで使う

```sh
npm run dev      # 開発モード（HMR）で起動
```

アプリ起動後:
- `Cmd+O` で対象フォルダを選択（通常は `mdv <path>` で起動するため、アプリ内で開き直す時に使う）
- 左ツリー（☰ で開閉）から Markdown ファイルを選んで閲覧
- ファイル閲覧中にツールバーの「☑ 差分」トグルで、そのファイルの git 差分（変更ハイライト）に切り替え

動作確認用のサンプルは `samples/` にあります。

## ビルド

```sh
npm run build      # 本番ビルド（out/ に出力）
npm run start      # ビルド済みアプリのプレビュー
npm run typecheck  # 型チェック
```

## 構成

- `src/main/` — Electron メインプロセス（ウィンドウ生成、IPC、ファイル走査、`git diff`）
- `src/preload/` — `contextBridge` で `window.api` を公開（`contextIsolation` 有効）
- `src/renderer/` — React + Vite の UI（Markdown レンダリング、Mermaid、差分表示）
- `src/shared/ipc.ts` — 3 プロセス共有の IPC 契約（チャンネル名・型）

Mermaid は ER 図の SVG DOM 構造が安定している `~10.9` 系に固定しています。
